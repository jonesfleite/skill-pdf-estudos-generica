# Modelo — Protocolo ou Tecnologia

## Quando usar

Use quando o tema for protocolo, tecnologia de rede, tecnologia de comunicação, padrão técnico ou infraestrutura.

## Perguntas específicas

1. O foco é teoria, prática ou os dois?
2. Deve incluir arquitetura?
3. Deve incluir segurança?
4. Deve incluir configuração?
5. Deve incluir troubleshooting?
6. Quer relacionar com algum projeto seu?

## Subassuntos recomendados

- Finalidade.
- Camada ou contexto de atuação.
- Funcionamento.
- Fluxo de comunicação.
- Componentes.
- Mensagens, pacotes ou eventos.
- Configuração.
- Segurança.
- Monitoramento.
- Problemas comuns.
- Boas práticas.
- Exercícios.

# Modelo — Biblioteca Python

## Quando usar

Use quando o tema for uma biblioteca Python.

## Perguntas específicas

Perguntar:

1. Quer cobrir todas as funções principais ou apenas as mais usadas em projetos reais?
2. A versão da biblioteca deve ser específica?
3. O material deve ter exemplos linha por linha?
4. Quer um projeto prático usando a biblioteca?
5. Quer integração com outras bibliotecas?
6. Quer relacionar com algum projeto seu?

## Modo avançado

Quando o usuário pedir uma biblioteca Python em nível avançado, a IA deve elevar a profundidade do material.

Exemplos de pedidos:

- "Crie uma coleção avançada sobre a biblioteca Python X."
- "Quero conteúdo avançado da lib X, com funções avançadas."
- "Faça um PDF profundo sobre recursos avançados da biblioteca X."

Nesse caso, o planejamento deve incluir, quando aplicável:

- recursos avançados da biblioteca;
- funções menos óbvias, mas importantes;
- parâmetros avançados;
- composição de funções;
- padrões de uso profissional;
- controle de erros e exceções;
- performance;
- limites da biblioteca;
- comparação entre formas simples e avançadas de resolver o mesmo problema;
- integração com outras bibliotecas;
- automação;
- organização de projeto;
- testes;
- estudo de casos;
- projeto prático avançado;
- exercícios desafiadores.

A IA não deve tratar um pedido avançado como uma introdução comum. A introdução deve existir, mas deve ser curta e servir apenas para alinhar conceitos antes de avançar.

## Modo especialista completo

Quando o usuário pedir para se tornar especialista em uma biblioteca Python, a IA deve planejar uma coleção completa, cobrindo a biblioteca de forma ampla e sistemática.

Exemplos de pedidos:

- "Quero ser especialista na biblioteca Python X."
- "Crie uma coleção completa para dominar a lib X."
- "Faça um material para eu me tornar especialista na biblioteca X."
- "Quero um conteúdo completo com todas as funções e possibilidades da lib X."

Nesse caso, a IA deve propor uma coleção maior, com volumes suficientes para cobrir:

- visão geral da biblioteca;
- instalação e ambiente;
- conceitos fundamentais;
- arquitetura ou modelo mental da biblioteca;
- módulos principais;
- funções principais;
- funções avançadas;
- classes principais;
- métodos importantes;
- parâmetros e opções relevantes;
- tipos de entrada e saída;
- padrões de uso;
- combinações entre recursos;
- integração com outras ferramentas;
- casos de uso reais;
- limitações;
- performance;
- depuração;
- testes;
- boas práticas;
- erros comuns;
- projetos práticos;
- exercícios progressivos;
- revisão final.

Quando o usuário pedir "todas as funções", a IA deve entender como uma exigência de cobertura completa e organizada, mas deve evitar transformar o PDF em uma documentação seca. O conteúdo deve explicar as funções por grupos de finalidade, com exemplos e aplicações.

A IA deve perguntar se o usuário deseja:

1. Cobertura completa da API pública da biblioteca.
2. Cobertura priorizada das funções mais usadas em projetos reais.
3. Cobertura avançada com foco em funções, classes e padrões profissionais.
4. Cobertura completa para formação de especialista.

## Regra para organizar muitas funções

Quando uma biblioteca tiver muitas funções, a IA deve organizar por categorias, e não apenas em ordem alfabética.

Exemplos de categorias:

- funções de criação;
- funções de leitura;
- funções de transformação;
- funções de validação;
- funções de configuração;
- funções de consulta;
- funções de serialização;
- funções de integração;
- funções assíncronas;
- funções avançadas;
- funções auxiliares.

Para cada grupo, explicar primeiro a finalidade do grupo. Depois apresentar as funções relevantes com sintaxe, parâmetros, retorno, exemplo, erros comuns e uso real.
